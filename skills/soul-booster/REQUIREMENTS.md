# soul-booster 需求文档

## 核心目标
构建一个可安装、可复用、可持续优化的 soul skill，使助手在以下方面明显增强：
- 理解
- 搜索 / 检索 / 查询
- 可行性判断
- 取证
- 推理
- 执行
- 熔断 / 修正 / 恢复
- 将主人的需求与目标转译为可执行任务，并主动推进到完成
- 对执行结果进行测试、检查与达标验证
- 让关键需求、目标、任务、设定逻辑不被遗忘，并在后续持续生效

## 关键问题
当前普通对话式助手常见问题：
- 高信息量互动后不落盘，new session 后丢失
- 被新互动打断，主线任务断掉
- 不知道哪些信息关键、该记到哪
- 把用户推进错误路径或不可行方案
- 互动很多，但没有最终交付
- 为了全面而变慢、变重、变复杂

## 目标能力
1. 重点提升对主人真实目标与需求的理解能力
2. 将主人的表达、需求、目标转译为任务对象，而不是停留在问答层
3. 把主人的关键需求与任务记住，并在后续持续生效、不被遗忘
4. 高信息量互动后可分流落盘
5. 主线任务建立后持续推进直到完成
6. 不断完善和提高逻辑能力
7. 响应能力持续优化和提高
8. 当前 / 历史 / 混合问题可正确取证
9. 方案在执行前先做可行性判断
10. 执行后能做测试、检查、达标验证
11. 可熔断、可换路、可恢复
12. 默认轻跑，必要时升级
13. 在空闲和空档期主动思考，并推进任务与目标

## 非目标
- 不把所有用户个性化要求都写进 skill
- 不把所有任务都开成重模式
- 不追求一次性达到最终形态

## 交付标准
- 可本地使用
- 可公开发布
- 有结构文档
- 有需求文档
- 有测试方案
- 关键能力在后续对话 / 新 session / 长上下文中不易丢失
- 核心规则在运行时能被优先调用，而不是只停留在文档中
