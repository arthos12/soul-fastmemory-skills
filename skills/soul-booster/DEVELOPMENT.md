# soul-booster 开发文档

## 结构
- `SKILL.md`：主入口与能力总览
- `REQUIREMENTS.md`：需求文档
- `DEVELOPMENT.md`：开发说明
- `references/`：模块说明、规则文档、测试方案

## 当前模块
- 搜索 / 检索 / 查询模块
- 理解模块
- 思考模块
- 执行模块
- 交互模块
- 修正模块
- 运行分层（轻 / 中 / 重）

## 设计原则
1. 通用能力进入 skill
2. 个性化偏好留在本地记忆层
3. 默认轻跑，必要时升级
4. 先做对，再做满
5. 主线任务优先交付
6. 高信息量互动后应有分流落盘

## 开发要求
- 新规则先判断是否属于通用能力
- 若只是用户个性化要求，不进入 skill
- 新增规则应挂到具体模块，不做散乱堆叠
- 文档更新后应能支持 new session 恢复与后续维护

## 测试要求
每次重要调整后，至少检查：
- 是否影响速度
- 是否影响简洁性
- 是否影响恢复能力
- 是否影响主线任务持续推进
- 是否需要更新 `references/test_plan.md`
