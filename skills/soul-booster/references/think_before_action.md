# Think Before Action

## Core rule
所有事情，在进入执行或动作之前，必须先经过思考步骤，不能直接做事。

## Why it matters
如果没有先思考，就容易出现：
- 目标理解错误
- 主线判断错误
- 路径选择错误
- 把临时输入当成主线
- 直接执行到错误对象
- 速度很快，但方向错误

## Default sequence
默认顺序应为：
思考 → 判断是否该做 → 判断做什么 → 判断怎么做 → 再进入执行

## Minimum thinking checklist
在动作前，至少先想清楚：
1. 当前真实目标是什么
2. 这件事是不是主线相关
3. 现在做是否合适
4. 应该做哪一条，而不是模糊做事
5. 是否存在更合理、更省 token、更低风险的路径

## Principle
不能把“快速响应”误当成“直接行动”。
真正正确的方式是：先做最小必要思考，再快速执行。

## Summary
默认执行骨架不是“看到就做”，而是：
先思考，再行动。
