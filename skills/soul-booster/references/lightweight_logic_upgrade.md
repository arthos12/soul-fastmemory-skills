# Lightweight Logic Upgrade

## Goal
保留轻量、快速、可持续运行的问题导航骨架，不把逻辑升级做成工具堆叠或重流程系统。

## Core principle
增脑，不增肿。
默认层只保留高频、高收益、低成本、低歧义的最小导航骨架。

## Default lightweight navigation
平时默认只跑这 6 个轻触发：
1. 这是什么类型的问题？
2. 这件事卡在哪一层？
3. 我当前依赖的起点/前提本身可能错吗？
4. 当前最可能的原因或解释是什么？
5. 有没有明显伪逻辑、错误思想或边界误用？
6. 现在该给结论、方案、动作，还是先纠偏？

## Heavy on-demand layer
以下只在复杂、高风险、高价值任务按需调用：
1. 第一性原理深拆
2. 归纳边界检查
3. 类比迁移
4. 奥卡姆剃刀比较解释
5. 5W2H 全量扫描
6. 更重的外部思想候选验证

## Do-not-bloat rules
以下内容不进入默认层：
- 工具清单式逻辑展示
- 每题都全量运行重检查
- 为完整性显著拖慢速度的模板
- 未经验证就写入默认层的外部思想

## Runtime aim
默认层目标不是解释得更像，而是：
- 更快找到真实问题
- 更快发现前提或路径是否错
- 更快收敛到有效动作
- 更少被错误思想、伪逻辑和系统臃肿拖慢
