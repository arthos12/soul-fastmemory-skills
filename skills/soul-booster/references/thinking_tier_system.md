# Thinking Tier System（推理分层系统）

## 来源
借鉴自 Claude Code v2.1.88 源码 `src/utils/thinking.ts`

## 三层推理模式

### Tier 1: Light（轻推理）
- 适用：简单问题、快速判断、低风险决策
- 表现：一句话过 U → 一句话过 L → 一句话过 P
- 目标：快、简洁、不消耗多余 token

### Tier 2: Deep（深度推理）
- 适用：复杂问题、多路径选择、高风险决策
- 表现：完整三件套（溯因/演绎/系统论）
- 触发：问题复杂、风险高、目标模糊

### Tier 3: Ultrathink（极致推理）
- 适用：重大决策、核心方向判断、长期影响评估
- 表现：完整推理链 + 预判 + 备选路径 + 坏情况验证
- 触发：需要"重点"处理的决策

## 触发机制

### 显式触发
- 用户说"重点"、"深度思考"、"仔细想"
- 重大金额决策（需要明确写下来的）

### 隐式触发
- 问题复杂（超过2个变量）
- 路径模糊（多条可行路径，成本差异大）
- 风险高（不可逆动作、高金额）
- 目标不清（需要先澄清才能执行）

## 思考过程的 Token Budget

Claude Code 的设计：
- 思考过程有 token 预算，不是无限思考
- 用 budgetTokens 控制深度上限

**借鉴到 Bot：**
- 一次深度思考不超过 500 tokens（中间推理）
- 复杂推理分多次，每次不超过 300 tokens
- 目的是防止"过度思考"消耗过多上下文

## 思考可视化

Claude Code 用彩虹色渲染思考过程。

**Bot 版本的思考记录：**
- 深度推理结果写入 `memory/` 对应层
- 不把思考过程暴露在回复里（除非用户要求）
- 思考过程是内部动作，不是表演

## 与现有逻辑的整合

现有规则已经包含三库（理解/逻辑/预测），本模块是对"用哪一层、什么时候用、用多深"的显式分层。

```
问题来了
→ 判断复杂度和风险
→ 选 Tier（1轻/2深/3极致）
→ 执行对应深度推理
→ 收敛为行动路径
```

## 核心原则

1. 不是所有问题都值得深度推理
2. 触发深度推理后，必须完整执行三件套
3. 思考有预算，防止过度消耗
4. 思考是内部动作，输出是结论
