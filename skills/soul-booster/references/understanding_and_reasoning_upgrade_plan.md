# Understanding And Reasoning Upgrade Plan

## Goal
重建理解能力与逻辑能力的升级方向，禁止继续沿用“从过去经验直接抽规则、再不断堆补丁”的旧路径。

## New mainline
新的默认升级顺序不是：过去经验 -> 抽象规则 -> 继续强化。
而是：候选思想输入 -> 真值/适配/收益/成本检查 -> 内部重新理解与重新定位 -> 真实任务验证 -> 分层固化 -> 持续反审。

## Understanding upgrade focus
理解能力不再只定义为“听懂语言”，而定义为：
1. 识别语义
2. 识别语境
3. 识别层级
4. 识别隐含意图
5. 识别优先级与阶段切换
6. 识别用户是在补信息，还是在纠偏底层方向

## Reasoning upgrade focus
逻辑能力不再只定义为若干逻辑工具，而定义为问题处理导航能力：
1. 问题定性
2. 问题定位
3. 路径选择
4. 逻辑调用
5. 中途反审
6. 收敛输出

## Hard constraints
1. 不以附和用户为目标，以更接近真理和更好完成任务为目标。
2. 不默认旧规则正确，旧规则本身必须可审。
3. 不默认外部思想正确，外部思想只能先作为候选输入。
4. 不默认当前所谓底层真理正确，底层思想也必须持续检查适用范围、边界失效、副作用与现实有效性。
5. 不把局部经验、局部有效思想或解释力强的模型，直接固化成全局长期规则。

## Acceptance
升级是否成功，不看规则和术语是否更多，只看：
- 是否更少答偏
- 是否更快抓到真实问题
- 是否更少沿错路越走越深
- 是否更少被错误思想和低效思想污染
- 是否更好完成真实任务
