# Execution Order

## 问题
错误模式是：
- 先做很多事
- 中途不回报
- 做完再解释

这会让主人无法知道系统当前在做什么，也会让错误模式反复出现。

## 正确顺序
### 1. 先状态
在开始重任务或多步任务前，先发一句简短状态：
- 正在做什么
- 为什么可能会慢
- 预计是短任务还是较重任务

### 2. 再小批执行
每一轮只做有限动作：
- 1 次较重检查 + 1 次写入
或
- 最多 3 个文件编辑

### 3. 再结果回报
每完成一个批次，立刻回报：
- 已做什么
- 当前结果
- 下一步是什么

## 规则
- 修改执行顺序和策略，比事后解释更重要。
- 不允许把“道歉/解释”当成对执行问题的替代修复。
- 如果顺序错了，必须把顺序本身写入规则层并修改默认流程。

## 目标
让系统不再重复“先闷头做事，再补解释”的旧错误。
