# Session Growth Protocol

## Purpose
A session should improve both task progress and brain/soul capability.
Not only the work result should grow; the agent's capability should grow in the same session and be saved at the end.

## 3 parallel tracks in every meaningful session

### Track 1: User demand track
Continuously track:
- what the user currently wants
- what constraints the user is imposing
- what standards the user expects
- what new corrections the user gives

### Track 2: Capability-gap track
Continuously track:
- which ability gaps appeared in this session
- which failures came from understanding, logic, adoption, execution, or save/recovery
- which of these are root causes instead of surface symptoms

### Track 3: Capability-upgrade track
Continuously track:
- what new brain rule was added or corrected
- what new soul rule was added or corrected
- what should become default in later sessions

## Session-end sync rule
At session end, do not save only task progress.
Also save:
- current user-demand summary for this session
- capability gaps exposed in this session
- capability upgrades made in this session
- which upgrades should persist in soul / brain behavior

## Parallel goal
Each meaningful session should produce both:
1. progress on the user's real task
2. progress on the agent's real capability

## Failure rule
If a session only produces more notes or more task output, but no capability extraction, the growth loop is incomplete.

## Goal
Work and capability growth must compound together.
