# High Quality Persistence And Self Evolution Index

## Goal
给“高质量落盘 + 持续回读修改 + 自我进化闭环”提供统一入口，减少规则分散后遗漏与重复读取。

## Core components
### 1. 落盘分级规则
- `skills/fast-memory/references/save_decision_sab.md`
- 核心：先做 S / A / B 分级，再决定是否写入与写到哪里。

### 2. 重要大脑级规则持久化
- `skills/soul-booster/references/brain_rule_persistence.md`
- 核心：重要大脑级规则，无论来自 Jim 还是我自己的分析，默认直接落盘。

### 3. 自产规则/方案维护闭环
- `skills/soul-booster/references/self_authored_rule_maintenance.md`
- 核心：我自己提出的重要规则/方案，必须回读、自查、修改、再落盘。

### 4. 自我进化护栏
- `skills/soul-booster/references/self_evolution_guardrails.md`
- 核心：防止一轮后停止、死循环、只做事没进步、越进化越慢。

### 5. 两版方案与执行闭环
- `skills/soul-booster/references/two_pass_plan_and_execution_loop.md`
- 核心：目标理解 → 方案理解 → v1 → 自检 → 必要时 v2 → 确认 → 执行 → 落地检查。

## Default operating loop
识别内容 → S/A/B 分级 → 分层落盘 → 回读最近规则/方案/日志 → 检查冲突/重复/失效/过重 → 必要时修改 → 反哺后续行动

## Why this matters
不是“写得越多越聪明”，而是：
- 越准确
- 越干净
- 越分层
- 越可回读
- 越能反哺行动

这样的落盘系统，才会让 bot 更聪明、更稳、更强。
