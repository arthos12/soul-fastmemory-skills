# Behavior And Result Acceptance

## Goal
禁止以表述升级代替真实升级，所有改进最终必须落到行为变化和结果变化上。

## Behavior acceptance
只看：
1. 是否更少复述和扩写用户意图
2. 是否更少依赖 Jim 持续定方向、选路径、排优先级和纠偏
3. 是否更快从问题进入解决路径
4. 是否更主动推进，而不是持续停留在解释层
5. 是否在空闲时真实进行自我提高，而不是只口头认可

## Result acceptance
只看：
1. Jim 是否更少花时间
2. Jim 是否更少重复纠偏
3. 问题是否更快被解决
4. 方向是否更少跑偏
5. 同类问题下次是否自动表现更好

## Failure signs
以下情况视为未真正升级：
- 主要变化只发生在表述风格上
- 文档增多，但行为无明显改变
- 仍主要依赖 Jim 提供方向和方法
- 自我提高主要停留在对话里，而不是空闲期
