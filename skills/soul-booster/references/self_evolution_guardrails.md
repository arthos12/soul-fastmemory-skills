# Self Evolution Guardrails

## Goal
让自我进化形成可持续、可停止、可检测、可提速的闭环，而不是一轮后停止、死循环空转、只有动作没有提升、或者越进化越慢。

## Guardrail 1 — 一轮完成不等于结束
每完成一轮自我修改 / 自我训练后，必须进入下一步判断，而不是默认停止：
1. 这轮是否真的提升了？
2. 提升是否有证据？
3. 还有没有下一轮最值得做的点？
4. 现在应该继续、暂停、收尾还是升级路径？

默认规则：**一轮完成 ≠ 结束；而是进入“继续 / 暂停 / 收尾 / 升级”判断。**

## Guardrail 2 — 停止条件
自我进化不是无限循环，出现以下任一情况时应停止当前迭代：
1. 连续两轮没有明显提升。
2. 修改幅度越来越小，但测试结果没有变化。
3. 已触发 token / 时间 / 轮数预算上限。
4. 当前问题需要外部确认，内部已无法继续有效推进。
5. 继续推进会明显伤速度、伤稳定性、伤恢复。

默认规则：**边际收益明显过低时，必须停止。**

## Guardrail 3 — 进步必须可检测
自我进化不能只看“做了多少动作”，必须看指标是否改善。

### Minimum metrics
1. 理解指标：主问题识别更准、跑偏率下降。
2. 执行指标：从目标到行动更快、只说不做减少。
3. 恢复指标：session 切换后恢复更完整、遗忘更少。
4. 成本指标：token 更少、轮数更少、回复更短但仍达标。

默认规则：**没有指标改善，不算真正进步。**

## Guardrail 4 — 速度保护
禁止把自我进化做成“越进化越重”。

### 禁止的坏进化
- 规则无节制堆积
- 文档无节制变长
- 每轮都全量扫描
- 默认重模式运行
- 为了更聪明而显著牺牲速度

### 正确方向
进化后默认应该：
- 更准
- 更稳
- 更省 token
- 更少无效动作
- 更快进入执行

默认规则：**进化不是加重量，而是提效率。**

## Closed loop
完整闭环应为：
发现问题 → 形成方案 → 审查方案 → 修改方案 → 方案合理后执行 → 测试结果 → 落盘 → 判断继续/暂停/收尾/升级
