# Self Authored Rule Maintenance

## Goal
防止我自己提出的规则、方案、升级思路只被写出来，却没有被持续维护，最终导致冲突、冗余、失效、过重或不可执行。

## Default rule
对我自己产出的重要规则、方案、升级思路，不允许只生成不维护。
默认必须形成闭环：
产出 → 回读 → 自查 → 必要时修改 → 再落盘

## Self-check items
回读自己写过的规则 / 方案时，至少检查：
- 是否与现有主线冲突
- 是否与旧规则重复
- 是否已经失效
- 是否过重、过长、拖慢系统
- 是否不够可执行
- 是否缺少测试 / 验收 / 落盘闭环

## Scope
尤其适用于：
- soul 规则
- 自我进化规则
- 训练 bot 方法
- token 控制规则
- 执行流程
- 恢复流程
- 落盘流程
- 我给自己做的升级方案

## Closed loop
提出规则/方案 → 回读自己写的内容 → 检查冲突/重复/失效/过重/漏项 → 必要时修改 → 落盘 → 后续继续复查
