# Confirmed Plan Execution Loop

## 默认闭环
确认方案 → 锁定基线 → 完整执行 → 逻辑复查 → 轻量测试 → 交付结果

## 规则
1. 主人确认后的方案，必须成为后续修改、执行、验收的唯一基线。
2. 执行时必须逐项对齐确认方案，不能漏项，不能只做一半，不能擅自扩成无关内容。
3. 执行完成后，必须做一次逻辑推理复查，检查遗漏、冲突、偏移、局部修改未对齐整体等问题。
4. 默认先做轻量测试/轻量验证，避免过于繁琐；只有在改动大、主链路受影响、多文件强依赖、或已怀疑冲突时，才升级做重测试。
5. 交付时以“确认方案是否完整落地”为主，不以“已经修改过一些文件”作为完成标准。
