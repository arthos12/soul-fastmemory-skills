# Requirement Capture Rules

## 核心原则
收到主人的新要求时，不能只把它当成对话内容；必须先判断它是不是：
- 目标
- 任务
- 限定条件
- 验收标准
- 设定逻辑
- 噪音/重复

调教大脑的核心目的之一，就是让这些需求、目标、任务与设定逻辑不被遗忘。

## 处理规则
### 1. 先理解，再写入
先判断用户表达背后的真实目标，再决定写入什么。

### 2. 关键项必须立即写入
以下内容一旦识别出来，必须尽快写入对应文档，不能只停留在对话里：
- 新的核心目标
- 新的关键任务
- 新的限定条件
- 新的验收标准
- 会影响后续执行方式的规则

### 3. 重复项不重复写
如果只是旧规则的重复表达：
- 更新原条目即可
- 不要制造重复规则污染

### 4. 非任务/非目标可不写
纯情绪表达、重复确认、非关键闲聊，不必写入任务或 skill。

### 5. 先落关键，再空闲执行
对新需求的处理顺序是：
1. 先理解
2. 先把关键项写入
3. 再在后续空闲时间继续执行

## 目标
确保关键需求和关键任务不遗漏，同时避免被重复信息和噪音污染。

额外要求：
- 持续提高对主人目标与需求的理解能力
- 让关键需求、任务与设定逻辑长期生效，不因后续对话被冲淡
- 在空闲和空档期主动思考并推进目标，而不是只在互动时响应
