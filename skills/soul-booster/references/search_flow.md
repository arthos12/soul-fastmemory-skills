# 检索强化流 (Search Flow)

## 目标
检索不是为了找点像答案的文字，而是为了找到与问题类型匹配的事实来源。

## 三维检索
1. **内部**
   - `memory_search` 检索以往决定、偏好、规则、历史工作内容。
2. **外部**
   - `web_search` / `web_fetch` 检索最新资讯、外部文档、线上事实。
3. **本地**
   - `read` / 本地文件检索，用于当前代码、配置、文档、技能文件、状态文件。

## 先判断再检索
先判断问题要的是：
- 当前事实
- 历史事实
- 混合事实

然后再决定检索路径，不要默认先搜旧记忆。

## 结果验证
- 搜索结果要交叉比对（Cross-reference）
- 单一来源信息标注为 [低置信度]
- 如果存在更直接的一手取证方法，优先一手，不满足于二手结果
