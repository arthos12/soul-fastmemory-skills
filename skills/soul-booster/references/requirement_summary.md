# Requirement Summary

## 1. 核心目标
- 重点提升对主人的目标、需求、任务的理解能力。
- 将需求/目标自动转成任务，并持续推进到完成。
- 关键需求、任务、设定逻辑不能被遗忘，后续必须持续生效。
- 调教大脑的目的，不是更会聊天，而是更会做事、做完、做好、记住、下次接上。

## 2. 执行目标
- 不再停留在问答型；要进入任务驱动型。
- 主线任务未完成前持续推进，完成并验收后才移出。
- 在交互时、空闲时、空档期都应继续推进目标。
- 高频互动时，非当前问题可暂缓，避免打断前台体验。

## 3. 记忆与恢复目标
- 高信息量互动后，关键内容必须分流落盘。
- 新 session / 中断后，应恢复主线、断点、规则与下一步。
- 关键需求、目标、任务、设定逻辑不能在后续对话中被冲淡或遗忘。

## 4. 取证与判断目标
- 当前事实必须实时核验，历史问题查历史，混合问题拆开取证。
- 不只要查，还要用正确方法查到真实数据。
- 执行前先判断方案可行性，避免把用户推进死路。

## 5. 运行与响应目标
- 前台优先：重任务前先做负载预告。
- 并发要稳定：三段式并发、小批次推进、用户插话即回前台。
- 并发要资源感知：受 CPU / 内存 / 上下文负荷约束，接近上限主动降载。
- 响应能力和执行速度都应持续优化，但不能以牺牲正确性为代价。

## 6. 测试与验收目标
- 任务结束标准不是“做过了”，而是“达标了”。
- 必须持续测试与检查结果，不达标继续改。
- 两个 skill（`soul-booster` / `fast-memory`）都要真实落地，不能只是聊明白。
- 练出来的理解力、任务化能力、持续推进能力、恢复能力，后续不能轻易丢失。
- 核心能力必须在后续运行中持续生效，而不是只在本轮对话表现好。
