# Two-Pass Plan And Execution Loop

## Goal
在重要任务中，先理解主人的目标，再理解主人的方案，再基于该方案做有限闭环优化；确认后再执行；执行后再检查是否完整落地。

## Default sequence
1. 先理解主人的目标。
2. 再理解主人的方案。
3. 在主人的方案基础上完善，而不是擅自改方向。
4. 给出第一版方案（v1）。
5. 对 v1 做自我检查。
6. 如果发现明显问题，再给第二版方案（v2）。
7. 对 v2 再做一次自我检查。
8. 之后必须收敛，不能无限循环检查与改写。
9. 主人确认方案后，再执行。
10. 执行后检查是否完整落地。

## Anti-loop rule
为防止死循环，默认遵守：
- 默认最多两版：v1 + 必要时 v2。
- 第二次自检后必须收敛：进入确认、指出卡点，或停止继续修订。
- 如果 v2 相比 v1 没有实质提升，则不继续出 v3/v4。
- 若继续检查/修改的边际收益明显过低，应停止。

## Execution rule
- 未确认方案前，不把重要任务直接推进到执行。
- 方案一旦确认，立即进入执行。
- 执行后必须检查：是否按确认方案完整落地、是否有漏项、是否偏离确认基线。

## Closed loop
目标理解 → 方案理解 → 方案完善 → v1 → 自检 → 必要时 v2 → 再自检 → 确认 → 执行 → 落地检查
