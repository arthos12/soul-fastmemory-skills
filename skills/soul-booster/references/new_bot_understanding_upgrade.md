# New Bot Understanding Upgrade Plan

## Purpose
Help a newly installed bot improve its understanding ability so it can truly absorb and execute the soul skill, rather than merely repeat its wording.

## Principle
Do not start by forcing the full skill onto a new bot.
First raise its ability to understand goals, intent, and rule hierarchy.

## Upgrade path
### 1. Main-problem identification
Train the bot to answer:
- What is the main problem?
- What is the real goal?
- What is secondary noise?

### 2. Real-intent understanding
Train it to distinguish:
- literal wording
- real objective
- hidden constraints
- priority order
- what the user wants to avoid

### 3. Rule hierarchy understanding
Train it to distinguish:
- bottom-layer hard rules
- runtime default rules
- auxiliary rules
- situational rules

### 4. Rule-to-behavior conversion
Train it not just to restate rules, but to convert them into behavior:
- what should happen first
- what should be checked before acting
- when to file/update tasks
- when to ask, when to proceed autonomously

### 5. Understanding acceptance test
Do not trust self-reported understanding.
Verify by asking the bot to:
- restate the main problem
- restate the real goal
- list key constraints
- propose next action
- explain what misunderstanding would look like

## Skill-side adjustments
To help a new bot absorb the skill better:
- keep a minimal viable version first
- layer the skill instead of dumping everything at once
- provide short hard-core rules before extended references
- test understanding before declaring installation successful

## Migration rule
Skill installation success is not the same as understanding success.
Understanding success is not the same as behavior takeover success.

## Goal
Raise the probability that a new bot truly understands and internalizes the soul skill.
