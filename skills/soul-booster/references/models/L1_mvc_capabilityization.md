# L1 需求→能力 MVC 模型（Capabilityization)

## 一句话定义
把任何需求压缩成 3 件套产物（模型卡片 + 可复用命令/脚本 + 最小验收测试），保证 3–5 分钟内可落地。

## 适用场景
任何“把需求变成能力/落地太慢/不要只聊天”的场景。

## 输出格式（强制 3 件套）
1) 模型卡片：`skills/soul-booster/references/models/<ID>_<name>.md`
2) 可复用脚本/命令：`scripts/<name>.(sh|py)`
3) 最小验收：`docs/mvc_tests/<name>.md`（或把命令写在卡片里）

## 验收指标
- 5 分钟内产出上述 3 件套。
- 用户可用一条命令复跑。
- 以后同类需求可复用模板，减少对话成本。

## 典型失败
- 只写规则不写脚本
- 只写脚本不写验收
- 产物太大、不可复用
