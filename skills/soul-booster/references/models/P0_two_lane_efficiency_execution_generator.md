# P0 两条线提效执行结构生成器（Two-lane Efficiency Execution Generator）

## 一句话定义
当用户提出“用两条线提高效率”（线1交付、线2验证/纠错）时，必须自动生成可运行的并行流水线结构：每条线的输入/动作/产出/验收，以及两条线的交接与失败回路。

## 适用场景
- 用户明确提“两条线/双线/并行”来提升效率与执行力。

## 强制输出（必须按表格/要点给出）
### 线1：交付线（Delivery Lane）
- 输入：待补完清单（[m]列表/优先级）
- 动作：每轮补完1个 [m]→[x]（稳定 done 增长）；必要时新增 [m]
- 产出：可运行产物 + done 增量
- 验收：1条命令 + 产物文件/关键输出

### 线2：验证/纠错线（Validation & Repair Lane）
- 输入：线1产物的验收命令列表 + 最近错误复发清单
- 动作：并发跑验证（不排队，不等窗口）+ 对失败项做最小修复并写防复发
- 产出：通过/失败清单 + 最小修复PR/提交 + 防复发闸门
- 验收：并发验证报告（通过率/失败项/下一步）

### 两条线的接口（必须写清）
- 线1交付产物 → 线2拿到验收命令并发验证
- 线2失败 → 反馈给线1（修复同一项，不新增新项）

## 硬规则
- 验证非阻塞：线2并发验证，线1不等线2跑完才开始下一条（但失败项必须优先返修）。
- WIP 限制：线1默认WIP=1（先稳定done增长），线2可并发验证多个。
- Direction优先：两条线只服务P0主线，禁止把并发扩展到业务/预测导致跑偏。

## 最小验收
用户问“两条线怎么提效”，输出必须包含：
- 两条线各自的输入/动作/产出/验收
- 接口与失败回路
