# T1 问题类型识别闸门（Question-Type Gate）

## 一句话定义
在出方案/执行/给清单之前，先判定用户此刻要的“问题类型”。类型不匹配直接 FAIL 重答，防止抓不住重点。

## 只允许三类（必须选其一）
1) **定位/解释**：用户在问“哪里错了/为什么/根因是什么/怎么判断”。
2) **方案+执行**：用户要你“给方案→自审→执行”。
3) **证据汇报**：用户要“完成比例/做了什么/给证据/可复跑”。

## 闸门规则（强制）
- 先输出一行：`Type: <定位|方案执行|证据汇报>`
- 若 Type=定位/解释：禁止让用户在 1/2/3 里选；必须直接回答“错误环节/根因/如何修正”。
- 若 Type=方案+执行：必须走 J0→A1→执行。
- 若 Type=证据汇报：必须给文件/命令/数值，不给计划。

## 一票否决
- 用户问“哪里出错了”，你却给“接下来做什么/让用户选项” → FAIL。
- 用户问“现在做”，你却给解释不执行 → FAIL。

## 验收
抽测 5 条：每条先判类型，再输出匹配的回答；若出现一次错配即不通过。
