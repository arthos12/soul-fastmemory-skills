# R0 根因优先模型（Root-cause First）

## 一句话定义
先找共因根因，再修复；禁止症状驱动的零碎补丁。

## 适用场景
任何出现“效率低/跑偏/死循环/反复修补”的问题。

## 固定步骤（不可跳）
1) **分层定位**：目标函数层 / 主次层 / 方案选择层 / 执行工具层。
2) **找共因**：能同时解释>=2个症状的原因。
3) **最小共因修复**：产物化（模型卡片/脚本/指标）。
4) **用指标验收**：必须在 metrics/decisions 中留下证据。

## 输出格式（强制）
- 症状（<=3条）
- 根因层级（1条）
- 共因（1条）
- 修复产物（文件清单）
- 验收命令（1条）
