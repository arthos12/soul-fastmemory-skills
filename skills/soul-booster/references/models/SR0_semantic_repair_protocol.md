# SR0 语义误解纠偏协议（Semantic Repair Protocol）

## 一句话定义
当用户指出“你没懂/跑偏/没抓重点/说人话/纠错不对”时，必须立刻按可验收方式修复语义理解，而不是继续解释。

## 触发词（任一命中即触发）
- `你没懂` / `没抓住重点` / `跑偏` / `纠错能力不对` / `说人话` / `理解错了`

## 强制动作（必须按顺序执行）
1) **复述用户此刻真正要的东西（1句）**
   - 形式："你现在要我回答的是：____"（只填一个）
2) **给修正后的答案（直接命中）**
   - 不给选项、不反问、不讲过程。
3) **指出我错的环节（1句）**
   - 只允许：问题类型识别错 / 主矛盾抓错 / 输出格式错 / 证据口径错。
4) **落盘防复发（最小改动）**
   - 在对应闸门/规则里加一条“一票否决/触发词”或补一个最小测试。

## 禁止
- 继续解释旧答案
- 让用户在 1/2/3 里选（除非用户明确要求做决策）
- 用术语堆砌替代“修正后的答案”

## 验收
- 用户说一次“没抓住重点”后，下一条必须能通过用户的“对/不对”判断。
