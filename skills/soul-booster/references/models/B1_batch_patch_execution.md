# B1 批量修改执行模型（Batch Patch Execution）

## 一句话定义
每次纠偏先出“修改方案（文件级变更清单）”，再一次性批量落地；禁止一个点一个点零碎改。

## 适用场景
用户指出问题并要求修复/提效/纠偏时。

## 输出格式（强制）
1) 方案（Plan）：
- 目标（1句）
- 变更清单（<=5条）：每条 = 文件/模块 + 要改什么 + 为什么
- 风险点（<=3条）
- 验收命令（1条）
2) 执行（Batch Apply）：
- 一次性修改所有列出的文件
- 修改后立刻给“证据清单”（progress_report/metrics）

## 验收指标
- 单次纠偏的有效产出密度提升（减少往返）
- 返工率下降
