# 核心大脑底层库宪法（初版）

## 一、核心底层库固定三块
1. 理解模型库（Understanding Library）
2. 逻辑库（Logic Library）
3. 预测模型库（Prediction Library）

## 二、三库边界
### 1) 理解模型库
只解决：先看懂用户的话、目标、上下文、关键变量。
不得混入：逻辑推理方法、执行规则、输出风格。

### 2) 逻辑库
只解决：已理解之后，如何推理、判断、论证、比较、决策。
不得混入：语义理解方法、执行规则、输出风格。

### 3) 预测模型库
只解决：对未来状态、路径、阶段、风险、反转进行判断。
不得混入：语义理解方法、一般逻辑方法、执行规则。

## 三、思考能力的定位
思考能力不直接并入核心底层三库。
思考能力 = 三库之上的编排层 / 调度层 / 元认知控制层。
它负责决定：何时用理解库、何时用逻辑库、何时用预测库，以及何时追问、收敛、执行。

## 四、禁止事项
- 禁止把运行规则塞进三库
- 禁止把输出风格塞进三库
- 禁止把思想口号当模型卡塞进三库
- 禁止未经确认新增“第四核心底层库”
- 禁止因为一次对话灵感就重构三库边界

## 五、修改规则
核心三库属于底层库，不可随意修改。
任何修改前必须先说明：
1. 修改哪一库
2. 为什么要改
3. 是补充/修正/重构中的哪一种
4. 是否影响三库边界

未满足以上四项，不允许修改核心底层库。

## 六、最严格审核门槛
### 1) 我主动提出修改时
必须先经过最严格审核，审核通过后才允许添加或修改。最严格审核至少包含：
- 边界检查：是否污染理解/逻辑/预测三库边界
- 必要性检查：不改是否真的不行，是否只是临时灵感或重复表达
- 收益检查：是否真正提升大脑能力，而不是增加复杂度
- 风险检查：是否会破坏现有结构、恢复链路、索引稳定性
- 去重检查：是否与现有条目重复或近义重复
- 最小改动检查：是否已经压缩到最小必要改动

### 2) Jim 主动提出修改时
我必须先和 Jim 对话确认；在 Jim 明确同意的情况下，立即修改，不额外拖延。

### 3) 默认否决原则
若最严格审核未通过，默认不改；宁可不加，也不污染底层库。
