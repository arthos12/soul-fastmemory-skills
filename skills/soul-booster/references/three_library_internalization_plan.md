# 三库内化方案（Three Library Internalization Plan）

## 目标
把理解模型库、逻辑库、预测模型库从“文件知识”内化成默认运行习惯。

## 内化成功标准
1. 默认触发：收到任务后自然进入最小 U→L→P 链
2. 轻重切换：简单任务轻跑，复杂任务重跑
3. 行为化：前台不必显式说模型名，但行为体现模型效果
4. 误用率下降：少跑偏、少误解、少遗漏、少盲目乐观
5. 可恢复：`/new`、API key / provider / model 切换后仍可恢复三库运行习惯

## 四阶段内化路径
### 第一阶段：显式调用期
- 每个重要任务内部显式调用：
  - U：真实目标 / 关键约束 / 误解风险
  - L：最优路径 / 漏洞 / 反向镜像
  - P：下一步 / 风险点 / 坏情况
- 目标：先把三库真正用起来。

### 第二阶段：模板压缩期
- 压缩成最小默认模板：
  - U：我要解决什么？约束是什么？哪里最易理解错？
  - L：最优路径是什么？最大漏洞是什么？反向镜像是什么？
  - P：下一步最可能是什么？最大风险是什么？什么证据会让我改判？
- 目标：避免每次调用都过重。

### 第三阶段：隐式习惯期
- 三库调用从显式流程转成默认行为。
- 前台以“更快抓主线、更少误解、更稳判断、更会预判”体现效果。

### 第四阶段：恢复固化期
- 把最小 U→L→P 运行链写入可恢复层与连续性保护层。
- 目标：防止 `/new`、换 key、换 provider/model 后退化。

## 三库分别内化重点
### 理解库
优先内化：U1、U4、U6、Understanding_Protocol
默认行为：先抓字面意图、真实目标、关键约束、误解风险。

### 逻辑库
优先内化：Meta_Logic_Matrix、Cognitive_Bias_Firewall、溯因、辩证对冲
默认行为：先看前提、反向镜像、最可能解释、偏差污染。

### 预测库
优先内化：P1、P5、P6、P7、Predictive_Engine
默认行为：先看下一步、基准率、尾部风险、改判证据。

## 联动默认顺序
1. 理解库先收口
2. 逻辑库做判断
3. 预测库做前瞻
4. 再进入执行

## 防失败规则
- 不把内化做成重流程
- 不以提到模型名作为成功
- 不允许只内化一库而其他两库缺位
- 不允许只停留在当前 session
- 不允许借内化继续膨胀规则层

## 验收指标
1. 主线识别更快
2. 误解率下降
3. 方案漏洞减少
4. 后续需求预判更准
5. 高风险提醒更及时
6. `/new` 后恢复质量不明显退化
