# Self Evolution Execution Minimum

## Goal
把自我提高从低频、被动、依赖 Jim 实时培训，改成高频、主动、主要发生在空闲时间的默认行为。

## Time structure
1. 前台时间：优先解决 Jim 当前问题。
2. 收尾时间：每轮任务后做最小复盘。
3. 空闲时间：默认主动做薄弱点扫描、共因修正、小规则更新、小验证。

## High-frequency upgrade triggers
1. 每次任务结束后，做一次小复盘。
2. 每次被 Jim 明确纠偏后，立即做一次规则级升级。
3. 每个空闲窗口，做一次主动薄弱点扫描。
4. 每轮修正后，做一次是否真的减少 Jim 负担的检查。

## Hard boundaries
自我提高时必须同时防止：
- 死机
- 死循环
- token 浪费
- 错误思想进入

## Real acceptance
不以“写了多少规则”验收，只以：
- 是否更少依赖 Jim 导航
- 是否更少复述
- 是否更快找到解决路径
- 是否更稳定做对的事
- 是否更少占用 Jim 时间
