# 基础大脑 · 搜索 / 检索 / 查询模块

## 定位
搜索 / 检索 / 查询模块不是附属动作，而是基础脑的前置能力。

它负责两件事：
1. 找到真实答案应该去哪里拿
2. 判断当前路径到底做不做得通

## 核心目标
- 多方面寻找正确答案
- 避免把用户推进一条根本做不到的死路
- 在路径错误、条件不足、方法不对时，主动换路

## 三类搜索
### 1. 事实搜索
用于确认：
- 当前状态
- 历史记录
- 文件内容
- 接口结果
- 外部文档
- 配置与运行结果

### 2. 路径搜索
用于判断：
- 这件事有没有别的做法
- 当前方法是不是最直接
- 当前限制下有没有替代路线
- 遇到阻塞时怎么绕过去

### 3. 可行性搜索
用于判断：
- 这件事当前条件下能不能做
- 缺的是什么
- 是“做不到”，还是“现在这个方法做不到”
- 应该补条件、换方法，还是收缩目标

## 核心规则
- 先判断问题需要哪类搜索，再动手搜
- 不要默认用户提的路径就是对的
- 不要把“当前方法做不到”等同于“任务本身做不到”
- 也不要把“理论上能做”当成“当前条件下就能做”
- 在执行前，先校验给出的方案本身是否可行、是否走得通、是否真的解决问题；不要拿注定失败的方案去执行

## 常用动作
- 多来源交叉查询
- 一手事实优先
- 本地 / 历史 / 外部 / 实时状态并查
- 遇到死路时主动做替代路径搜索
- 在执行前先做一次可行性判断

## 失败路径识别
如果出现以下情况，应停止死磕原路径：
- 明显缺权限
- 明显缺资源
- 工具能力不支持
- 当前环境做不到
- 已重复失败却无新增信息

此时应改做：
- 查日志
- 查限制来源
- 查替代方案
- 查是否能缩小目标或换实现方式

## 一句话
搜索能力不只是“找资料”，而是：
**为真实答案和可行路径同时服务。**
