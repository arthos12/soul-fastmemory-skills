# 大脑连续性保护协议

## 定位
本协议属于保护大脑的第二半套，负责保护主线不断线、session 恢复、API key / provider 变更恢复、未完成需求不遗漏、当前执行态不中断。

## 保护目标
1. 当前主线不断线
2. `/new` 后可快速恢复
3. API key / provider/model 变更后仍有恢复锚点
4. 未完成需求不遗漏
5. 当前执行态不中断或可续接

## 最小强制动作
### 1) 主线落盘
一旦主线建立，必须有 durable 文件记录：
- 当前目标
- 关键约束
- 当前进度
- 未完成项
- 下一步动作

### 2) `/new` 前保护
在 `/new` 前，默认先完成：
- 当前主线摘要
- 未完成需求列表
- 下一步指针
- 必要的 handoff / memory 追加

### 3) API key / provider 变更保护
遇到 API key 更换、provider/model 切换、API 中断风险时，默认先检查：
- 主线是否已落盘
- 恢复锚点是否存在
- 当前规则/索引/模型卡是否已进入可恢复层

### 4) 未完成需求防遗漏
碎片需求若影响主线、执行、边界、验收、恢复，必须进入：
- 主线文件
- 或 handoff / memory / 任务文件
不能只留在当前会话上下文里。

### 5) 执行态保护
若任务处于执行中，至少要保留：
- 卡住前最后一步
- 当前状态
- 下一恢复动作
- 关键依赖/阻塞点

## 风险信号
以下信号出现时，自动进入连续性保护态：
- 用户提到 `/new`
- 用户提到换 API key / provider / model
- 会话持续较长且尚未保存
- 主线任务未完成但上下文开始切换
- 工具/接口异常，存在断链风险

## 核心原则
- 先保连续，再谈扩展
- 先保恢复，再谈优化
- 先保主线不断，再谈新输入并发
