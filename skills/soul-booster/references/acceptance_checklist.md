# Acceptance Checklist

## 目标
用于检查 soul-booster 是否已满足当前核心需求。

## A. 理解与任务化
- [ ] 能优先理解主人的真实目标，而不是停留在字面问答
- [ ] 能把需求/目标转成任务对象
- [ ] 能把补充需求、限定条件识别为任务/约束/验收标准

## B. 记忆与持续生效
- [ ] 关键需求、目标、任务、设定逻辑已写入可恢复文档
- [ ] 后续对话中不会因新消息把关键要求冲淡
- [ ] 高信息量后能正确分流落盘

## C. 执行与推进
- [ ] 主线任务建立后持续推进，未完成不移出
- [ ] 高频互动时不打断前台体验
- [ ] 空闲/空档期会继续推进目标

## D. 响应与速度
- [ ] 重任务前有负载预告
- [ ] 执行顺序遵守：先状态 → 小批执行 → 结果回报
- [ ] 高负载时优先跑核心规则，不被冗余规则拖慢

## E. 判断与取证
- [ ] 当前事实实时核验
- [ ] 历史/混合问题走正确取证路径
- [ ] 执行前会做方案可行性判断

## F. 验收
- [ ] 任务结束标准是达标，不是做过
- [ ] 有测试与检查机制
- [ ] 不达标会继续修正
