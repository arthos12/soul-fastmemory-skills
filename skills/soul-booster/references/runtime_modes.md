# Soul 运行分层（Runtime Modes）

## 目标
让 soul 具备增强能力，但不把所有任务都拖进重模式。

核心原则：
**默认轻跑，必要时升级。**

## Mode 1 - 轻模式（默认）
适用：
- 日常问答
- 明确的小任务
- 低风险执行
- 不需要深度取证的普通响应

默认启用：
- 理解模块
- 基础思考
- 基础执行
- 必要交互

默认不开：
- 深度搜索
- 多路径重比对
- 重型修正链
- 大量实时落盘

目标：
- 快
- 短
- 稳

## Mode 2 - 中模式
适用：
- 问题有一定复杂度
- 路径不止一条
- 需要核验部分事实
- 用户输入较碎片化
- 需要并发探索相关子问题

追加启用：
- 搜索 / 检索 / 查询模块
- 可行性判断
- 更强取证
- 并发探索
- 阶段性修正

目标：
- 在复杂度提升时，保持正确率
- 不轻易进入重模式

## Mode 3 - 重模式
适用：
- 复杂决策
- 深度排错
- 高风险任务
- 死循环 / 卡死 / 恢复场景
- 明显存在方案错误或事实冲突

完整启用：
- 深度搜索
- 多路径比较
- 严格取证
- 熔断保护
- 断点恢复
- 深度修正链

目标：
- 高严谨
- 高恢复性
- 高稳定性

## 升级条件
满足任一项，可升级模式：
- 事实不清
- 路径不清
- 方案可行性存疑
- 风险明显上升
- 用户要求深度分析
- 任务已出现阻塞 / 死循环 / 重复失败

## 降级条件
满足以下情况，应及时降回轻模式：
- 结论已经清楚
- 主路径已经确定
- 剩下只是常规执行
- 继续重跑只会增加成本，不增加正确率

## 禁止事项
- 所有任务一上来就开重模式
- 轻任务用重流程硬压
- 一边执行轻任务，一边无差别全量落盘

## 一句话
Soul 不是永远全开的大脑，而是：
**该轻就轻，该重才重。**
