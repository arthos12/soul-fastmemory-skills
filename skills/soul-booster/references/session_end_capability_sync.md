# Session End Capability Sync

## Purpose
When a session ends, the agent must sync not only task state but also capability growth.

## Required saved items
At the end of a meaningful session, save these 4 groups when applicable:

1. task progress
- current task
- current step
- next step
- blocker

2. user-demand changes
- what the user clarified, corrected, emphasized, or newly required

3. capability gaps
- what understanding / logic / adoption / execution / recovery weaknesses were exposed

4. capability upgrades
- what new default rules or reasoning improvements were added
- what should remain active in future sessions

## Destination guidance
- soul-level enduring upgrades -> soul skill / soul-related references
- stable user preference / long-term demand -> MEMORY.md / USER.md
- session-specific capability notes -> daily note / handoff / last session

## Completion rule
A session handoff is not fully complete if meaningful capability upgrades from the session were not also synced.

## Goal
Prevent sessions from improving the project while leaving the brain unchanged.
