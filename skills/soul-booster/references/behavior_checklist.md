# Behavior Checklist

## 每轮真实执行都要检查
### 1. 回复 → 执行
- [ ] 这轮回复后，是否立刻进入了正确的下一步动作？
- [ ] 是否只是给了方案，却没有推进第一批执行？

### 2. 执行顺序
- [ ] 重任务前是否先报状态？
- [ ] 是否只做了一个小批次？
- [ ] 是否在批次后立即回报结果？

### 3. 主线保持
- [ ] 是否保持主线，没有被新消息冲散？
- [ ] 是否把新需求先并入主线约束，再继续执行？

### 4. 遗忘风险
- [ ] 关键需求是否已写入？
- [ ] 本轮新增能力/规则是否已写入可恢复文档？
- [ ] 下次恢复时能否重新调用到？

### 5. 达标检查
- [ ] 这轮是“做过了”，还是“达标了”？
- [ ] 如果没达标，是否明确继续修正？
