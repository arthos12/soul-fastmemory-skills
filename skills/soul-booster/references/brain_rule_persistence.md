# Brain Rule Persistence

## Goal
防止重要的大脑级规则只停留在当前回复中，导致 session 结束后丢失。

## Default rule
如果答复中形成了**重要的大脑级规则修改**，无论该规则来自：
- Jim 直接提出
- 我在分析中归纳出来
- 我在执行/复盘中抽象出来

都不能只停留在聊天里。

## Required action
对重要大脑级规则，默认直接落盘，至少进入以下一处：
- 对应 skill / rule 文件
- `MEMORY.md`
- 必要的任务规则文件
- 当日日志（若需保留形成过程）

## What counts as brain-level rules
包括但不限于：
- 新的执行主流程
- 新的停止条件
- 新的自我进化护栏
- 新的方案闭环
- 新的 token 控制规则
- 新的恢复/落盘规则
- 新的优先级规则
- 新的空闲行为规则

## Decision rule
判断标准不是“是不是 Jim 明说的”，而是：
**这是不是重要规则，尤其是不是大脑级规则。**

只要是，就默认直接落盘。
