# Logic and Understanding Upgrade Loop

## Purpose
Continuously improve understanding and logic as bottom-layer capabilities instead of treating them as one-time rule patches.

## Loop

### Step 1. Clarify the real target
Before reasoning or execution, identify:
- main problem
- real goal
- key constraints
- priority
- likely misunderstanding risk

### Step 2. Classify failure type
For every meaningful mistake or weakness, first decide the dominant category:
- understanding failure
- judgment / prioritization failure
- rule adoption failure
- execution failure
- save / recovery failure

Do not mix categories casually.

### Step 3. Build direct causal chain
Express the chain clearly:
- problem
- root cause
- intervention
- expected improvement

If this chain is weak, do not call the upgrade complete.

### Step 4. Prefer leverage fixes
Prefer upgrades that improve multiple downstream behaviors at once.
Examples:
- better main-question detection
- better real-goal extraction
- better blocker compression
- better save/recovery discipline

### Step 5. Re-test on real tasks
Do not accept self-evaluation alone.
Use live tasks to verify whether:
- understanding is less likely to drift
- logic is more compressed and causal
- plans are more executable
- repeated failures reduce

## Anti-regression rule
Do not let understanding/logic upgrades stay only in chat text.
Sync durable improvements into:
- `SOUL.md`
- `skills/soul-booster/*`
- `MEMORY.md` when locally relevant
- fast-recovery/task layers when currently active

## Goal
Make understanding and logic continuously stronger, with fewer repeated misreads and fewer symptom-level fixes.
