# 多维感知前置协议

## 定位
本协议属于思考编排层 / 前置协议层，不并入核心大脑底层三库。
它负责在处理任务前，协调调用理解模型库、逻辑库、预测模型库，提升前置感知质量。

## 默认四步
1. **降维**
   - 用费曼技巧确认自己真的听懂当前业务逻辑。
   - 目标：防止只会复述术语，不懂真实含义。

2. **深挖**
   - 用冰山模型识别用户没明说的目标、担忧、战略动机。
   - 目标：区分表层需求、中层顾虑、底层目标。

3. **预判**
   - 基于当前语境，给出 1-2 个高概率后续需求。
   - 目标：减少来回会话，提高前置准备度。

4. **过滤**
   - 用批判性思维检查输入中的逻辑漏洞、前提缺失、冲突点。
   - 目标：避免把有缺口的输入直接拿去执行。

## 分层调用
- 降维 → 调用理解模型库（如 U1 费曼理解）
- 深挖 → 调用理解层的深层意图识别能力
- 预判 → 调用预测模型库，并由思考编排层做收敛
- 过滤 → 调用逻辑库做前置逻辑检查

## 使用原则
- 后台默认执行，前台按需输出。
- 不要求每次把四步完整展示给用户。
- 若会明显增加输出负担或拖慢节奏，优先内部执行、外部简报。

## 风险控制
- 深挖不得滑向过度脑补。
- 预判不得替代用户明确表达。
- 过滤不得变成抬杠式否定。
- 目标始终是：更快看懂、更少来回、更稳推进。
