# Fast Memory Transfer Spec

## 目标
让安装 `fast-memory` 的其他 bot / agent，不只是拿到几个记忆相关文件，而是真正学会当前这套记忆系统的核心工作方式。

## 应继承的核心部分
1. 记忆分层：长期记忆 / 每日日志 / session handoff / checkpoint 分层存储。
2. 查询路径：先 search，后 get；优先小片段读取与回答，避免全量扫描。
3. 写入原则：只写长期有效、会反复使用、会影响决策的信息；避免流水账。
4. 防污染策略：分层、精简、清理，并配合准入阀门、去重阀门、过期阀门。
5. 恢复机制：新 session 先恢复 MEMORY / 最近 daily notes / handoff / checkpoint，再进入任务。
6. 收尾机制：一轮工作结束或暂停时，主动做 checkpoint、handoff、必要的任务/记忆更新。
7. 历史与当前区分：记忆用于历史背景与恢复，不直接充当当前事实。
8. 目标导向：记忆服务于理解、总结、执行与恢复，而不是囤积信息。

## 不迁移的部分
- Jim 的私有长期记忆内容本身。
- 只适用于当前本地环境的敏感配置、token、密钥、私人上下文。

## 判断标准
如果其他 bot 安装后，只会“存一些笔记”，却不会做分层、恢复、收尾、防污染与按需检索，说明迁移不完整。
