# Memory Enforcement Rules

## 目标
把 `fast-memory` 从“记忆说明文档”推进成“默认必须遵守的记忆运行规则”。

## 硬约束
1. 新 session 默认先恢复关键记忆层，不得完全等用户重讲。
2. 出现关键结论、关键参数、下一步、阻塞时，默认动态保存，不得全部依赖手动提醒。
3. 一轮工作结束或阶段暂停时，默认主动收尾，不得长期悬空。
4. 历史记忆不得直接冒充当前事实。
5. 长期记忆必须保持精简，不得把流水账无差别塞进 `MEMORY.md`。
6. 卡顿 / 中断 / freeze 风险出现时，必须先做诊断性落盘。

## 失败判定
若 bot 能复述记忆规则，但真实会话中不能稳定恢复、保存、收尾与防污染，则视为底层接管未生效。
