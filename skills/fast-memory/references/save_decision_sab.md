# Save Decision S / A / B

## Goal
把“要不要落盘”从模糊判断改成 S / A / B 三级快速决策，减少关键规则漏写、减少记忆污染、减少长期层无效膨胀。

## S 级：直接落盘
这些内容默认直接落盘，不等待额外确认：
- 大脑规则
- 系统偏好
- 执行红线
- 模型切换规则
- 空闲规则
- 停止条件
- 自我进化护栏
- 重要优先级规则
- 重要恢复规则
- 重要 token 控制规则
- 稳定长期有效的运行约束

默认去向：
- `MEMORY.md`
- 对应 skill / 核心规则文件
- 必要的任务规则文件

## A 级：先问是否记录
这些内容默认先问，再决定是否落盘：
- 方案
- 策略
- 任务
- 阶段性安排
- 某轮讨论形成的执行计划

默认去向（确认后）：
- `TASKS.md`
- `memory/YYYY-MM-DD.md`
- 对应 task file / project file

## B 级：默认不落盘
这些内容默认不写：
- 解释性对话
- 临时细节
- 一次性闲聊
- 无长期价值的过程性噪音

## Default rule
先分级，再决定写哪里；不要先模糊判断“要不要记”。

Closed loop:
识别内容类型 → 判定 S / A / B → 决定是否写入 → 选择落盘位置
