## 一、PM 套利系统的真实门槛
### 保留结论
- PM 上长期能吃到的钱，很多不是靠“看懂市场”，而是靠：
  - 数据采集
  - 市场/逻辑依赖检测
  - 约束或优化求解
  - 执行系统
  - 风险控制
- 单市场和跨市场都可能存在错价；真正难的不是“想到套利”，而是把它系统化、规模化、稳定化。
- 执行风险是硬门槛：算对了不等于赚到。非原子成交、滑点、盘口深度、延迟，都可能把理论 edge 吃掉。

### 对当前主线的实际意义
- 继续强化本地执行基础设施是对的。
- 延迟/漂移/执行风险必须进入策略与执行层，不是聊天备注。
- 若未来继续往 PM 套利系统深入，求解/约束层是潜在升级方向，但现在还没到先上复杂数学的时候。

### 不保留为真相的部分
- 外部文章中的利润规模、收益数字、数学名词，只能当启发，不当已验证事实。

## 二、Binance -> PM 时间差套利方向
### 保留结论
- 市场普遍流传并反复出现的高价值方向是：`Binance 先动，PM 后反应`。
- 如果这类机会存在，核心更像执行 alpha，而不是宏大叙事分析。
- 值得持续研究的不是故事，而是这些硬问题：
  1. Binance 价格变化到 PM 盘口变化的时间差
  2. 这段时间内可用流动性有多少
  3. 延迟/滑点会吃掉多少 edge
  4. 哪类 market / 哪个时间窗口更容易出现这种机会

### 对当前主线的实际意义
- 我们现在更应该做：
  - 记录 signal / confirm / submit / fill
  - 记录 drift bps
  - 记录不同时间窗口的反应差
  - 检查 edge 是否真能落到收益上
- 这也再次证明：执行层与延迟测试不是边角料，而是主线必做层。

### 不保留为真相的部分
- 外部流传的参数，比如某个固定涨跌阈值、某个固定秒数窗口，只能作为候选假设，不当已验证参数。
- “20 行代码赚很多钱”“每天稳赚”之类表达，一律按营销化压缩处理。

## 三、外部爽文/营销叙事的处理原则
### 保留结论
- 有些内容虽然信息密度低、营销味重，但反复出现同一个叙事方向，本身也能说明市场在关注什么。
- 对当前量化主线而言，真正值得保留的通常不是收益故事，而是：
  - 叙事背后反复出现的结构性方向
  - 市场参与者普遍相信哪里有 edge
  - 哪类机会说明竞争已经开始变强

### 当前统一处理方式
- 不直接备份原文链接和营销包装。
- 只保留：
  - 我提炼出的结构性结论
  - 可验证假设
  - 对当前执行主线的实际意义
- 所有外部内容默认分三层：
  1. `提炼结论`
  2. `候选假设`
  3. `已验证事实`

## 四、BTC 历史数据在当前主线中的作用
### 保留结论
- 如果继续做 BTC / CEX 策略，历史数据不是可选项，而是基础层。
- 历史数据的用途不只是回测，还包括：
  1. 验证策略是否只是短期巧合
  2. 检查不同市场阶段是否有效
  3. 校准交易频率 / 成本 / 回撤
  4. 为未来 `Binance -> PM` 联动研究打底

### 当前主线中的正确定位
- 历史数据是第一层，不是全部。
- 真要做短周期真钱策略，还必须叠加：
  - paper 验证
  - 延迟记录
  - 滑点/成本建模
  - 实时执行验证

### 当前动作含义
- 我们本地已有 `BTCUSDT 5m` 数据，可继续拿来筛 breakout / reversion 这两条 CEX 策略。
- 下一步应逐步补更长区间、更多 timeframe、更多分阶段统计。
