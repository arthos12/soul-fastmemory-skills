# Perception Protocol（感知协议）

目标：先感知正在做的事情中的高价值信号，再决定动作。感知优先级固定为：**风险 / 收益 / 异常**。

## P1 业务感知
- 数据异常：0价、字段错位、重复 marketId、realized/m2m 混淆
- 收益异常：胜率异常低、ROI不动、与预期偏差过大
- 方案差异：我和 BR 的入场位置、时间窗、价格口径、过滤条件差异

## P2 运行感知
- 什么任务正在运行
- 最近一次运行时间是否超时
- 状态文件是否持续更新
- 订单/结果/报告是否同步增长
- 是否被聊天/别的任务带偏导致主线停住

## P3 大脑/能力感知
- 漏问题、漏步骤、漏交付 = 重大问题，不是普通失误
- 一旦发现“漏了用户的点 / 只修一半 / 解释替代执行”，立即进入能力修复态

## 覆盖检查（回复前强制）
- 用户提了几个问题/要求？
- 我是否每个都：理解了 / 回答了 / 转成动作了？
- 若有任一未覆盖，禁止发送“已完成”式回复
