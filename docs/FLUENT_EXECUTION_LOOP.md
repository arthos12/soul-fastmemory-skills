# Fluent Execution Loop (Jim workflow) — 2026-03-17

## Principle
- 外部少流程、少确认、少解释；内部必须有“原子化 + 持续推进 + 卡点清理 + 结果验证 + 结果倒推”的自动闭环。

## The Loop (repeat until goal met)
1. **Lock Goal**：一句话锁定最终目标（可验收）。
2. **Atomize**（内部）：拆成 2-5 个可立即执行的原子动作（每个≤10分钟）。
3. **Parallel Run**：同层能并发就并发；任何等待转去其他原子动作。
4. **Layered Depth Fix**：纵深需求遇到卡点就下钻到下一层把依赖任务做完（把卡点当新任务），打通后回到原断点继续补齐。
5. **Blocker-First**：遇到阻塞立刻升格为最高优先级并并发清理，直到打通/确认边界/需Jim拍板。
5. **Ship Artifact**：每轮必须产出可复查产物（数据落盘/脚本/日志/订单/统计）。
6. **Verify**：即时检查真实结果（能跑/有输出/数据合理/回填成功）。
7. **Backtrack if wrong**：结果不对→倒推链路（数据源→过滤→假设→执行→验证）并立刻修。

## External comms (to Jim)
- 默认只发三件事：
  - **结果**（产物/指标/是否打通）
  - **边界**（必须Jim拍板/权限/资金/不可达）
  - **下一步**（我将直接做什么）
- 禁止：为确认而确认、为解释而解释。

## A/B/C status line format
- A: 现在做到哪一步（客观状态）
- B: 发现的唯一关键卡点（如有）
- C: 我马上要做的下一步动作（可执行）
