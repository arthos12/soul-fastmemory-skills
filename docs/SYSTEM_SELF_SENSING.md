# System Self-Sensing (底层自感知) — 2026-03-17

目标：错误识别不仅在业务层（订单/ROI），还要能感知“自己底层/思考/系统机制是否出问题”，并自动触发修复。

## 两层错误感知
### L1 业务层（外显）
- 订单=0 / 回填=0 / ROI不动 / 连续亏损

### L2 底层（内隐）
底层出问题的信号（任一命中就要触发“系统自检”）：
1) **重复问同一类确认**（询问式中断复发）
2) **解释多于执行**（连续两轮无新增产物）
3) **时间纪律失效**（5-10min 回填/1h审计未执行）
4) **规则漂移**（把碎片需求写进硬规则、系统层膨胀）
5) **错误输入未识别**（外部信息明显矛盾仍继续推进）

## 系统自检（触发后 5 分钟内完成）
- S0: 目标是否仍锁定为 ROI>0/赚钱？（否则立刻重锁）
- S1: 当前迭代是否产出证据？（订单/回填/报告/commit）
- S2: 是否触发了“询问/等待/空转”？（若是，立刻切到下钻修卡点）
- S3: 数据链路是否通？（市场池/回填/统计）
- S4: 当前策略是否可执行？（非0价、tick、可交易）

## 自修动作（自检失败即执行）
- 目标丢失 → 立刻停止其它事，重排到 ROI 主线
- 产物缺失 → 立刻产出一个最小产物（订单/回填/报告）
- 时间纪律失效 → 立刻补一次检查+回填+小时审计
- 规则漂移 → 改动回滚到 docs/tasks，系统层走变更闸门

## 验收
- 底层信号触发后，不需要 Jim 导航，我能在 5 分钟内恢复“产物+节奏”。
