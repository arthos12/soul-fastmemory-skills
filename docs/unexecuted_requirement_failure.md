# Unexecuted Requirement Failure

## Purpose
Define why a clear requirement can still fail to enter execution, and force direct correction.

## What counts as failure
A requirement has already clearly appeared, but no real execution started within the same working phase.

## Allowed root causes
1. atomization failure
2. priority distortion
3. rule over-completion illusion
4. surface-conversation drift
5. fake-ready state

## Correction rule
When this failure is detected:
1. name the failure type
2. convert requirement into task atoms immediately
3. execute the first task atom immediately
4. only after execution, decide whether any rule/document patch is still needed

## Rule-complete definition
A rule set is complete enough only when:
- it supports current execution directly
- it reduces repeat failure
- it no longer blocks the next real action
- it has minimum acceptance
- no more rule-writing is needed to start current work

If all 5 are true, further rule-writing counts as execution delay.
