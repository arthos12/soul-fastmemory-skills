# Capability Taxonomy v1（能力分类表）— 2026-03-17

目的：把“感知/复盘/时间/效率”等能力先归类，后续按类训练、按类验收、协同调用。

## L0 北极星
- 目标达成（ROI转正/赚钱）+ 低流量高产出

## L1 四大能力域

### C1 感知（Sensing）
- S1 业务感知：订单/回填/ROI/胜率/样本量/分桶变化
- S2 底层自感知：询问复发、解释>执行、时间纪律失效、规则漂移、外部矛盾未识别
- 验收：能在“变差的第一信号”出现时触发纠偏，而不是事后

### C2 复盘（Retrospective）
- R1 小复盘（1h）：产出/结果/时间纪律/卡点/资源/策略迭代 + Top3根因+修复动作+证据
- R2 大复盘（8h）：抓最影响目标的Top3根因，修复后1-2小时内应看到指标变化
- 验收：复盘必带修复、必带证据、下一轮指标可验证

### C3 时间观念（Time Discipline）
- T1 短周期节奏：5-10min 回填检查；到点必查，不等回填
- T2 小时审计：每小时胜率/ROI快照
- T3 迭代节奏：每小时至少1次可复查迭代产物
- 验收：节奏不中断；中断即触发底层自检

### C4 效率（Efficiency / Local-first）
- E1 本地优先：脚本跑业务、缓存增量、批量处理
- E2 模型预算：每小时最多一次模型调用（除非P0故障）
- E3 输出压缩：对外只报结果/边界/下一步
- 验收：流量下降但产物不下降（订单/回填/报告持续）

## L2 协同调用规则（把能力用起来）
- 感知触发复盘：任何红灯（订单=0/回填=0/ROI不动/底层故障信号）→ 立刻进入 5分钟系统自检 + 1h小复盘
- 复盘必须落到迭代：复盘结束必须产生至少1个迭代产物（参数/订单/回填/脚本/报告）
- 时间驱动节奏：5-10min & 1h & 8h 三层节奏并行；谁断谁是当前卡点
- 效率约束一切：能本地就本地；能批量就批量；能不问就不问
