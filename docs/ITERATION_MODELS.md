# Iteration Models (自我迭代模型库) — 2026-03-17

目标：更主动、更快、更省流量地把“结果”推到达标（赚钱/ROI转正），避免等待、空转、问话。

## M0 结果驱动闭环（默认常驻）
- 输入：目标 + 当前结果快照
- 动作：下单/回填/统计/改策略
- 输出：下一小时 ROI/胜率必须改善或明确“为何未改善”的可证伪原因

## M1 10分钟微循环（短周期交易专用）
- 触发：5m/10m/15m 市场
- 节奏：
  1) 生成订单
  2) 10分钟内检查结算/回填
  3) 若无订单/无回填/结果异常 → 立刻倒推（市场池/过滤/接口/脚本）并修

## M2 每小时“快照-改参”循环
- 输入：hourly_report.json（订单数/回填数/胜率/ROI/分桶）
- 动作：只允许做 1-3 个参数改动（阈值/过滤/市场池），并立刻开下一批订单
- 输出：下一小时对比快照

## M3 8小时根因复盘（只抓前三个）
- 输入：8小时内的小时快照序列 + 失败样本
- 动作：只提炼 3 个最影响目标的根因（数据源/策略/执行/验证/资源），每个根因只给 1 个最短修复动作并立刻执行
- 输出：修复后 1-2 小时内应看到指标变化，否则判为修复无效并换路

## M4 卡点下钻模型（纵深任务）
- 触发：出现“拿不到数据/接口不通/订单=0/回填断”
- 动作：把卡点当新任务下钻到依赖层做完（直到打通），再回到断点继续

## M5 流量预算模型（本地优先）
- 原则：本地脚本跑业务；模型只做推理和写代码
- 预算：每小时最多一次模型调用做策略调整；其余靠脚本产出报告

## M6 质量闸门（防自欺）
- 若出现以下任一项，立即视为故障并修：
  - 订单数=0（市场池/过滤/脚本问题）
  - 回填数=0（结果链路断）
  - 连续2小时 ROI 不改善且无可证伪原因（空转）
