# Retrospective Template (1h / 8h)

## 1h 小复盘（每小时一次）
- 产出：本小时新增了什么？（订单/回填/ROI快照/脚本/数据）
- 结果：胜率/ROI 是否改善？如果没有，为什么？
- 卡点：是否出现等待/空转/询问式中断？若有，立即改哪一条？
- 下一步：下一小时的 1-3 个最小动作是什么？

## 8h 复盘（每8小时一次）
- 目标：距离 ROI>0 / 赚钱还差什么？
- 三大问题：过去8小时最影响目标的 3 个问题是什么？（按影响排序）
- 修复动作：每个问题对应的 1 个最短修复动作（马上做）
- 规则沉淀：哪些应进入 docs/tasks，哪些需要进大脑层（触发条件+验收+边界）
