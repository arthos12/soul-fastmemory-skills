# Layered Task Model (纵深分层任务模型) — 2026-03-17

## When to use
- 需求纵深很深、存在嵌套依赖（A/B/C，其中 C 需要 D/E/F 才能补齐）。

## Core idea
- **同层并发**：同一层的元需求 A/B/C 可以并发推进。
- **遇到卡点就下钻**：把卡点视为“新任务”，立刻下钻到下一层（D/E/F）直到打通。
- **打通后回到断点**：下钻任务完成后，返回原层继续补齐 C，最终 A+B+C 完成即整体完成。

## Execution rule
1. 维护一个“任务栈/断点栈”（内部）：记录当前层、当前子目标、当前断点。
2. 每个原子动作≤10分钟；超时就拆更小。
3. 下钻优先级 > 继续解释；卡点 = 当前最高优先级。
4. 外部输出只给：结果/边界/下一步。

## Example (BR 数据)
- L0 目标：拿到 BR 页面/接口数据并完成分析
  - L1-A：拿到 positions
  - L1-B：拿到 pnl 曲线
  - L1-C：拿到 activity/更深明细
    - L2-D：定位真实接口
    - L2-E：分页/滚动触发
    - L2-F：落盘+解析
  - 当 L1-A/B/C 都完成 → L0 完成。
