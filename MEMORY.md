## 长期有效信息

- 用户明确区分“卷项目”和“投资”：
  - 卷项目需要和他人竞争，容易卷到死亡。
  - 投资重点是判断资产成长性、周期位置、时效性，以及认识自己和标的。
- 用户主张：持续完善投资逻辑，是更值得长期投入的道路。

  1. 以小博大
  2. 注意周期/时间投入
  3. 避免“卷”陷阱，强调时效性
  4. 复利
  5. 长期主义
  6. 抓现金流
- 用户重视芒格、巴菲特、段永平一类的长期主义、能力圈、风险控制、复利思维。

### 操作偏好
- 涉及 token / key / 密码 / 配置修改时，优先提供“一条命令 + 交互输入 + 自动写入”的方式。
- 用户很在意响应速度；长会话时要优先建议文件化归集、checkpoint、必要时切新会话。
- 用户在 new session 或工作中，可能会用“加载记忆 / 加载数据 / 恢复记忆 / 恢复数据 / 先加载一下 / 你先读一下”等宽松说法；只要明显是在要求恢复上下文或读取已有资料，就应主动执行加载，不必拘泥固定词条。
- 用户表达出“结束 session / 这轮先到这 / 先收尾 / 你去记一下 / 保存一下这次内容”等明确收尾意图时，都应视为 session 结束信号；无需拘泥固定触发词。在 Jim 的语境里，“结束 session”默认就等于：立即做 session 收尾、提炼关键信息并落盘保存，然后完成交接。此时应主动更新 checkpoint、SESSION_HANDOFF.md，以及必要的任务/记忆文件。

### 记忆系统策略
- 本地记忆负责存储/检索；大模型负责理解/总结/执行。
- 记忆查询遵循：先 search，后 get；以小片段读取和回答为主，避免全量扫描。
- 结束 session 时执行收尾落盘；加载记忆时优先读取 `MEMORY.md` + 最近日志/交接文件。

### 防污染 / 提速策略
- 目标：解决长期记忆垃圾信息过多、内容过长导致检索变慢、命中变差的问题。
- 本质：防污染策略就是提速策略；记忆越干净、越短、越分层，检索越快、命中越准。
- 分层：`MEMORY.md` 只存长期规则；日志只存当天细节。
- 精简：只写结论 / 下一步 / 约束，禁止流水账和冗余叙述。
- 清理：每周合并重复内容、标记过期信息、把关键规则迁移到长期记忆。
- 准入阀门：只有“长期有效 / 会反复使用 / 会影响决策”的信息，才允许进入 `MEMORY.md`；否则只进日志或不记录。
- 去重阀门：新信息写入前，先判断是否只是旧规则的重复表达；若是同义重复，优先更新原条目，不新增。
- 过期阀门：阶段性、临时性、一次性信息默认不进入长期记忆；若确需记录，应标注有效期或阶段属性。
- 结果：同时实现两件事——清理垃圾信息、提高回忆速度。
