# MEMORY.md

## 长期有效信息

### 用户与工作方式
- 用户当前主要关注：投资逻辑、项目判断、长期复利、现金流。
- 用户明确区分“卷项目”和“投资”：
  - 卷项目需要和他人竞争，容易卷到死亡。
  - 投资重点是判断资产成长性、周期位置、时效性，以及认识自己和标的。
- 用户主张：持续完善投资逻辑，是更值得长期投入的道路。

### 投资偏好 / 原则
- 投资优先检查项：
  1. 以小博大
  2. 注意周期/时间投入
  3. 避免“卷”陷阱，强调时效性
  4. 复利
  5. 长期主义
  6. 抓现金流
- 用户重视芒格、巴菲特、段永平一类的长期主义、能力圈、风险控制、复利思维。

### 关键系统文件
- 核心原则文件：`CORE_PRINCIPLES.md`
- Notion 目标文件：`notion/NOTION_TARGETS.md`
- Notion token 文件：`~/.config/notion/api_key`

### 操作偏好
- 涉及 token / key / 密码 / 配置修改时，优先提供“一条命令 + 交互输入 + 自动写入”的方式。
- 用户很在意响应速度；长会话时要优先建议文件化归集、checkpoint、必要时切新会话。
- 用户在 new session 中会用简短指令，如“加载记忆”或“加载数据”；此时应主动读取核心文件恢复状态。
- 用户表达出“结束 session / 这轮先到这 / 先收尾 / 你去记一下 / 保存一下这次内容”等明确收尾意图时，都应视为 session 结束信号；无需拘泥固定触发词。此时应主动执行交接：更新 checkpoint、SESSION_HANDOFF.md，以及必要的任务/记忆文件。
