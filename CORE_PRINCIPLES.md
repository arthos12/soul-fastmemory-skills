# CORE_PRINCIPLES

> 这份文件用于沉淀“做事的底层原则”，让你在评估项目/做取舍时有一套稳定的标准。
> 你是个人（单人操作），所以原则与流程会特别强调：**可执行、可验证、可持续（复利/长期主义）**。

## 快速索引（以后查这个就够）

### 原则（10条）
1) 清晰优先
2) 可验证（可观测/可复现/可回滚）
3) 最小可行（先闭环再扩展）
4) 风险评估（显式化，优先不可逆风险）
5) 复利思维 / 长期主义（沉淀资产：自动化/文档/组件/渠道/品牌）
6) 单人可操作（不依赖大团队/重资源/等别人）
7) 趋势与竞争（风口优先）：先看增长空间/趋势，再看竞争与自身条件
8) 小蓄（积蓄 / 积少成多）：可持续小优势 + 稳定积累，避免不成比例风险
9) 顺势而为：跟随大趋势，不逆势硬扛；趋势未明则降仓/降承诺
10) 周期 × 赔率（以小搏大）：先看生命周期/窗口期，再看收益率/赔率

### 芒格思维模型（映射到这套原则，作为评估清单）
- **反过来想（Invert）** → 对应 #4 风险评估：先列“怎么会死/怎么会亏大”，再决定是否做。
- **机会成本** → 对应 #6 单人可操作：你做这个，就不能做什么？是否值得占用你的主线精力？
- **能力圈** → 对应 #1 清晰优先 + #6 单人可操作：你是否真的理解关键变量？边界在哪里？
- **安全边际** → 对应 #4 风险评估：有没有留足时间/资金/仓位/价格的缓冲？
- **概率×期望值** → 对应 #2 可验证：用可观测数据估概率，算最坏/最好情景的期望值。
- **激励机制** → 对应 #4 风险评估：相关方激励是否会导致作恶/短视/造假？
- **认知偏差检查** → 对应 #2 可验证：有没有确认偏误/从众/过度自信？你有哪些反证？
- **第二层思维** → 对应 #7 趋势与竞争：别人会怎么做？竞争会如何升级？拥挤交易的后果？
- **Lollapalooza（多因素叠加）** → 对应 #5 复利/长期主义：有没有多种正向因素叠加形成复利？或多重风险叠加？
- **少犯错 + 耐心等大机会** → 对应 #5 复利/长期主义：避免大亏损，减少无意义交易/项目切换。

### 流程（你发项目 → 我按原则评判 → 给结论）
- 你提供：目标/成功标准/约束/用户与场景/分发路径/MVP/48小时验证/风险与回滚
- 我输出：原则打分(0-2)+理由、关键风险Top3、结论(继续/暂停/放弃)、48小时下一步
  - 其中 **#8 小蓄** 会额外检查：是否支持稳定积累/复利，是否为了短期爆发承担不成比例风险

## 0. 使用范围

- 适用：单人项目选择、需求取舍、功能优先级、执行计划、长期策略。
- 不适用：需要严格数学证明/法律/医疗等专业资质背书的场景（这里只提供决策框架）。

## 1. 核心原则（可持续补充/修改）

1) **清晰优先**：先把目标、约束、成功标准说清楚，再讨论方案。
2) **可验证**：尽量把结论落到可观测指标/可复现实验；方案要可回滚。
3) **最小可行**：先做最小闭环（MVP/原型/验证），再扩展。
4) **风险评估（显式化）**：把成本、依赖、失败模式写出来；优先处理不可逆风险。
5) **复利思维 / 长期主义**：优先做会持续产生收益的资产（自动化、文档、可复用组件、渠道/品牌沉淀）。
6) **单人可操作**：不做“必须等别人/必须大团队/必须重资源”才能推进的项目；把范围砍到你能独立闭环。
7) **趋势与竞争（风口优先）**：
   - **适用于“卷项目”**：优先选择高速增长、成长空间大的趋势/风口；在趋势成立的前提下，再评估竞争强度与自身条件，选择你能参与并胜出的细分切口。
   - 警惕：项目会“卷到死亡”（无尽迭代 + 红海竞争 + 体力战）。单人更要避免把自己拖进不可赢的消耗战。
8) **小蓄（积蓄 / 积少成多）**：优先追求可持续的小优势与稳定积累，避免为短期爆发去承担不成比例的风险；让系统长期可复利。
9) **顺势而为**：优先跟随大趋势与主要矛盾（市场趋势/行业趋势/自身优势），不逆势硬扛；趋势未明时降低仓位/降低承诺，等信号明确再加码。
10) **周期 × 赔率（以小搏大）**：评估项目先看“生命周期/窗口期能持续多久、你需要投入多久”；再看“收益率/赔率是否划算”。优先选择：投入可控、窗口足够、失败可承受但成功回报显著的项目（以小搏大）。

## 项目 vs 投资（你的区分）

- **卷项目**：核心是与他人竞争（产品、渠道、执行、供给）。要评估“是否有增长空间 + 你是否能在竞争中胜出”，否则容易卷到死亡。
- **投资**：核心是判断资产/行业的成长性与周期位置；同时要认识自己（能力圈、风险承受、纪律）与标的（商业/叙事/资金/风险）。
- 你的主张：与其把精力卷在项目的无尽竞争里，不如把时间投入到**持续完善投资逻辑**（可复用、可复利、长期有效）。

### 投资的原则（你给的版本，作为投资评估的优先检查项）

1) **以小博大**：才是增长最快的（投入可控，成功回报显著）。
2) **注意周期/时间投入**：花时间很少、赚很多的项目更好（时间成本极敏感）。
3) **避免“卷”陷阱，强调时效性**：投资不靠与人竞争取胜，更靠早期介入与时间点；不要把投资当成体力卷项目。
4) **复利**：让正确的系统与资产持续滚雪球。
5) **长期主义**：优先能长期成立的逻辑，而不是一次性机会。
6) **抓现金流**：优先能带来稳定现金流/可兑现收益的资产或路径。

## B. 快速决策规则（经验法则）

- **写不出成功标准**：先别做，先澄清。
- **48 小时做不出验证**：范围太大，继续砍。
- **写不出分发路径**：先别做产品，先做渠道/受众验证。
- **不可逆风险不清楚**：先做小规模、可回滚试点。
- **连续 2 周没有可见进展**：回到 MVP，删功能、做更小闭环。

## C. 单人每周复盘（15 分钟）

- 这周我只推进的“主项目”是什么？
- 我做了哪个“分发动作”？（发帖/私聊/上架/投放/冷邮件）
- 指标有没有动？（关注/咨询/注册/付费/留存）
- 最大阻力是什么？（技能/时间/渠道/产品/心态）
- 下周 48 小时内要完成的最小验证是什么？
