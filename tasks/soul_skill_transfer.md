# Task: Soul / Skill Transfer and Bot Capability Migration

## Goal
Build, solidify, publish, and validate a transferable capability stack centered on `soul-booster` and `fast-memory`, so other bots can install the skills and gain stronger understanding, reasoning, execution continuity, and session-to-session capability growth.

## Key facts / key requirements
- Bottom-layer abilities such as understanding, thinking, and logic should be solidified into `SOUL.md` and also updated into skill files, not left only in-session.
- My capability improvement must also improve the skill; otherwise the goal of training other bots fails.
- Final goal is not only updating the skill, but predicting and validating whether another bot can actually gain these abilities after installing it.
- The bot should strengthen autonomous work during idle windows and reduce the old mode where progress depends on renewed interaction.
- Tasks/goals should have durable files that describe goal, key facts, progress, and unresolved items; they must not live only inside the session.
- When goals/tasks become outdated or change, the file must be updated accordingly rather than leaving stale descriptions active.
- GitHub repo for publishing skills: `https://github.com/arthos12/soul-fastmemory-skills`.
