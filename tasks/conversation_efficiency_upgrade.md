# Conversation Efficiency Upgrade

## Goal
Reduce conversation count and token waste while increasing completed work, daily sustainable session count, and per-session output density.

## Locked acceptance
1. Each meaningful reply must advance at least one task atom.
2. Requirements must be atomized before execution.
3. Explanation-only rounds are invalid unless blocked by a real boundary.
4. More work should move into files/scripts/batch execution instead of staying in chat.
5. Session count down, completed work up.

## Current gaps found on self-check
- Rules existed, but trigger rate was not yet stable.
- Lack of a fixed pre-reply checklist made it easy to drift into explanation.
- Lack of a dedicated task board for conversation-efficiency work made requirements easy to remain in chat.
