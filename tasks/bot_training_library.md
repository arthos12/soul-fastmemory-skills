# Bot Training Library

## Goal
沉淀一套可复用的 bot 训练方法库，供 Jim 以后训练其他 bot 时直接调用；避免每次临时现想、重复试错、重复踩坑。

## Hard rule
**所有有效训练方法、命令模板、纠偏方式、适用场景与结果结论，必须存档。**
不允许只停留在聊天里。

## Library structure
### 1. 方法层
记录：
- 方法名称
- 适用 bot 类型
- 适用问题类型
- 训练目标
- 使用时机
- 风险/注意事项

### 2. 命令模板层
记录：
- 可直接发给 bot 的命令
- 命令版本
- 用途
- 适用条件
- 精简版 / 强化版 / 弱 bot 版

### 3. 日志层
记录：
- 训练对象
- 使用了哪条命令/方法
- 训练前问题
- 训练后表现
- 是否有效
- 是否需要继续纠偏

### 4. 纠偏层
记录：
- 常见失败模式
- 对应修正命令
- 哪类 bot 容易出现该问题

## Default use rule
当 Jim 说要“训练 bot / 给我训练命令 / 给我弱 bot 训练法 / 给我省 token 训练法”时，默认先查本库，再给当前最合适的方法与命令。

## Current direction
以后训练 bot 默认走两条并行路径：
1. **skill 路线**：通过 `soul-booster` + `fast-memory` 迁移底层能力
2. **命令路线**：通过可直接发送给 bot 的训练命令快速拉升行为与智力

两条路共同推进，而不是只靠其中一条。

## Training priority
以后训练 bot 的重点，不只是让它当前更会答，更重要的是训练它：
1. **自我进化能力**：能在真实任务后自查、自评、提炼问题、形成下一轮优化。
2. **空闲时自我进化能力**：能在空闲窗口主动做轻量复盘、能力升级、未完成需求检查、恢复链路检查，而不是空闲就完全停住。
3. **自我进化四大护栏**：防止做完一轮就停、死循环浪费 token、只有动作没有提升、以及越进化越慢。

默认把这三项视为训练 bot 的高优先级目标。
