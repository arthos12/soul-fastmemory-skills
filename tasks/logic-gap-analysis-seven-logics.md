# 七类逻辑在三库中的映射与漏项分析（执行稿）

## 用户提出的七类逻辑
1. 形式逻辑
2. 辩证逻辑
3. 结构逻辑
4. 系统逻辑
5. 概率逻辑
6. 演化逻辑
7. 逆向逻辑

## 当前结论（先锁定）
### 已有但需强化
- 概率逻辑：已部分在预测库（贝叶斯、基准率、胖尾）中存在，但输出标准化不足。
- 结构逻辑：已部分在理解层/表征层存在，但未明确硬化为逻辑母项。

### 有影子但未压实
- 逆向逻辑
- 系统逻辑
- 演化逻辑

### 明显漏项 / 高优先级候选
- 形式逻辑
- 系统逻辑
- 演化逻辑

### 值得补但需慎重硬化
- 辩证逻辑
- 逆向逻辑

## 映射（现有三库中已有什么）
### 形式逻辑
现有接近项：约束一致性、伪逻辑扫描、前提检查。
问题：缺少显式的命题/蕴含/充分必要/矛盾检测底层卡。

### 辩证逻辑
现有接近项：阶段变化、反转预警、正反馈与转折、部分易经变化边界。
问题：还没有硬化为可验证的“矛盾-转化-阶段切换”逻辑卡，容易滑向空泛解释。

### 结构逻辑
现有接近项：representation、schema mapping、contextual salience。
问题：更偏理解层，未把“结构决定约束/路径/结果”抽成逻辑母项。

### 系统逻辑
现有接近项：正反馈、风险预判、多维感知。
问题：缺少闭环、回路、耦合、延迟、二阶效应、系统边界等系统性底层逻辑卡。

### 概率逻辑
现有接近项：贝叶斯预测、基准率、胖尾/黑天鹅。
问题：置信区间、时间窗、失效条件、分布输出仍未标准化。

### 演化逻辑
现有接近项：周期、正反馈、变化/阶段判断、扩散候选。
问题：缺少路径依赖、选择压力、竞争扩散、阶段跃迁的统一母模型。

### 逆向逻辑
现有接近项：根因分析、失败树、最坏情况推理、方案自证/打脸点。
问题：这些分散存在，没有独立的底层逆向逻辑卡来统一“从错倒推/从失败反推/从结果反推条件”。

## 执行顺序（按优先级）
1. 先做“形式逻辑”候选卡的硬边界分析
2. 再做“系统逻辑”候选卡的硬边界分析
3. 再做“演化逻辑”候选卡的硬边界分析
4. 辩证逻辑与逆向逻辑放入慎审队列
5. 结构逻辑与概率逻辑优先做强化分析，不急着新增

## 重要边界
- 这里是“候选分析”，不是直接修改三库。
- 只有真理级底层逻辑模型，才能讨论是否进入逻辑库。
- 应用方法、工具模型、工作流、风格规则，一律不得混入。
