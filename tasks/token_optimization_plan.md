### 第二阶段：系统优化（本周） - 整合DeepSeek建议
#### 1. 数据输入优化（DeepSeek建议一）
- [ ] **技术指标替代原始K线**: 计算MA5、MA20、RSI等指标，只发送结果
- [ ] **压缩订单簿数据**: 只发送前3档，计算买卖压力比
- [ ] **定时摘要而非实时全量**: 价格波动>1%或RSI超买/超卖才调用AI

#### 2. 提示词工程优化（DeepSeek建议二）
- [ ] **明确约束输出格式**: 在system prompt中指定输出格式和长度限制
- [ ] **使用指令模板**: 定义多个场景的专用提示词
- [ ] **设置max_tokens**: API调用时设置较小值（10-50 tokens）

#### 3. 调用策略优化（DeepSeek建议三）
- [ ] **降频轮询**: 将高频调用改为低频，关键时段临时提高
- [ ] **合并多个分析请求**: 一次请求分析多个币种
- [ ] **设置调用冷却时间**: AI建议后5分钟内不重复调用同一品种

#### 4. 上下文管理优化（DeepSeek建议四）
- [ ] **无状态调用**: 大多数场景使用独立调用，不传历史
- [ ] **短期记忆仅保留关键决策**: 只保留最近1轮或手动摘要
- [ ] **定时重置会话**: 每10次调用或每小时清空历史

#### 5. 缓存机制优化（DeepSeek建议五）
- [ ] **本地缓存相同输入结果**: 1分钟内相同问题直接返回缓存
- [ ] **相似数据去重**: 价格变动<0.1%时沿用上次分析结果
