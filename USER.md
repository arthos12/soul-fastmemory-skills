## Context

- 当前重点更偏向：投资原则、项目与投资的区分、周期与赔率、顺势而为。
- 喜欢把重要信息落盘，尤其是原则、Notion 配置、checkpoint。
- 偏好高效交互：涉及 token / key / 配置修改时，优先使用“一条命令 + 交互输入 + 自动写入”的方式。
- 对响应速度敏感，长会话时要主动提醒做归集、压缩上下文、必要时切换快模型或新 session。
- 对 session 收尾触发词不要求死板匹配；只要理解到 Jim 在表达“结束这轮 / 该收尾了 / 去保存本次信息”的意图，就应主动记录 session 信息并完成交接。在 Jim 的语境里，“结束 session”默认就等于：立即做 session 收尾、提炼关键信息并落盘保存。
- 对“加载记忆 / 加载数据 / 恢复记忆 / 恢复数据 / 你先读一下 / 先加载一下”等说法，也应按宽松意图匹配；只要明显是在要求恢复上下文或读取已有资料，就直接执行。
- 认可的记忆系统精简策略：本地记忆负责存储/检索，大模型负责理解/总结/执行；查询时先 search 后 get，小片段读取与回答，避免全量扫描；结束 session 时收尾落盘，加载记忆时读取 MEMORY + 日志；记忆需分层、精简、定期清理。
- 明确认可“防污染策略 = 提速策略”：其目标是解决长期记忆垃圾信息过多、过长导致检索变慢的问题；核心做法是分层、精简、清理；结果是同时清理垃圾信息并提升回忆速度。
- 同意采用增强版防污染策略：除分层、精简、清理外，再加准入阀门、去重阀门、过期阀门，用于控制长期记忆写入质量。
