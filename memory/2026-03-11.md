## Distilled Rules

1. 结束 session = 默认触发收尾与落盘。
2. 加载记忆 / 加载数据 / 恢复记忆等 = 默认触发上下文恢复。
3. 触发规则以意图识别为主，不拘泥固定词条。
4. 未写入文件的信息不视为稳定可恢复记忆。
5. 以后记忆系统按这套规则执行。
6. Jim 给出一版精简系统策略：
   - 本地记忆负责存储/检索，大模型负责理解/总结/执行；
   - 先 search 后 get，小片段回答，禁止全量扫描；
   - 结束 session 收尾落盘；加载记忆读 MEMORY + 日志；
   - 分层：MEMORY.md 放长期规则，日志放当天细节；
   - 精简：只写结论/下一步/约束；
   - 清理：定期合并重复、标记过期、迁移关键规则。
7. Jim 进一步定义“防污染策略”的目标与本质：
   - 目标：解决长期记忆里垃圾信息多、内容过长导致检索慢的问题；
   - 核心三条：分层、精简、清理；
   - 本质：防污染策略就是提速策略；
   - 效果：同时清理垃圾信息、提高回忆速度。
8. Jim 同意采用增强版防污染策略，在原三条基础上再加入：
   - 准入阀门：只有长期有效 / 反复使用 / 影响决策的信息才进长期记忆；
   - 去重阀门：同义重复不新增，优先合并更新原条目；
   - 过期阀门：阶段性/临时性/一次性信息默认不进长期记忆，必要时标注有效期或阶段属性。
