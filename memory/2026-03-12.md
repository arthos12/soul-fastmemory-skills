## 2026-03-12 soul skill product definition
- Jim clarified a new open-source direction beyond fast-memory: build a real `soul skill` that can be published on GitHub and installed by others.
- Core split decided: the soul system should be divided into two modules:
  - `基础脑` / base brain = public, reusable cognitive/execution layer
  - `人格脑` / persona brain = user-trained personality layer
- Important conceptual rule: persona is not preset by the skill itself; personality is mainly formed by the user's long-term training, preferences, values, and corrections.
- The base brain's role is not “more chatty” but “more capable”: stronger reasoning, stronger search/retrieval, stronger memory use, faster response, better stability, interruption recovery, and less user decision load.

## 2026-03-12 base brain principles distilled with Jim
- Jim repeatedly tightened the intended style of the base brain. The useful condensed rules that matched his intent were:
  - 先完成，再修改。
  - 以完成任务为第一目标。
  - 先看目标，抓关键点，压掉小问题，先做出来，再修改。
  - 归纳总结和抽象不能偏离主人的目的和意图；可以做更合理的归类和整理。
  - 互动主要放在需求、目标描述、问题确认；主人不关心的内容默认直接处理。
  - 大纲就是逻辑；大型任务应按主次矛盾展示，当前执行详细，其余简短。
  - 能并发的任务应并发推进和并发完成，但主线必须清晰。
  - 小问题要归并到大问题，不持续输出主人不关心的技术噪音。
  - 上下文语义理解要强化：结合当前任务背景理解省略词和真实指向，避免脱离上下文的字面解释。
  - 如果前后表述冲突，要主动检查并选择更稳、更准、更贴近主人真实意图的版本；无法判断再问主人。
- A key correction from Jim: avoid over-abstracting into pretty but detached explanations. He explicitly preferred short, direct, operational statements that stay anchored to the real task and real intent.
