# NEW_SESSION_BOOT.md

## 目标
在 new session 中，尽快恢复工作状态，减少因聊天上下文清空造成的“失忆感”。

## 恢复顺序（按优先级）
1. `SOUL.md`
2. `USER.md`
3. `MEMORY.md`
4. `TASKS.md`
5. `CORE_PRINCIPLES.md`
6. `notion/NOTION_TARGETS.md`
7. 最新 checkpoint（优先读 `memory/2026-03-12-checkpoint.md`，之后再读最近两天 daily memory）

## 新 session 最短唤醒指令

### 推荐说法 1
读取长期记忆、原则文件、Notion targets、最新 checkpoint，然后继续工作。

### 推荐说法 2
加载 MEMORY.md、USER.md、TASKS.md、CORE_PRINCIPLES.md、notion/NOTION_TARGETS.md 和最新 checkpoint。

### 推荐说法 3（最短）
加载记忆并进入工作状态。

## 恢复后应该知道的事
- 用户叫 Jim
- 用户当前重点：投资逻辑、长期主义、复利、现金流
- 用户明确区分“卷项目”和“投资”
- 核心原则在 `CORE_PRINCIPLES.md`
- Notion 目标与 ID 在 `notion/NOTION_TARGETS.md`
- Notion token 在 `~/.config/notion/api_key`
- 长会话变慢时，优先做归集和 checkpoint，而不是依赖聊天上下文
- 配置或 token 输入时，优先给用户“一条命令 + 交互输入 + 自动写入”的方案

## 注意
- new session 会丢失聊天里的短期细节，但不会丢失已落盘的信息。
- 如果发现不知道当前状态，优先读取上述文件，不要凭空猜。
