## 量化系统检查（整合DeepSeek建议）

### API使用
- [ ] 量化脚本不使用LLM API ✅ 已确认
- [ ] 只使用数据API（Gamma、Binance等）
- [ ] 批量处理数据，减少API调用
- [ ] 合理使用缓存

### 数据输入优化（DeepSeek建议一）
- [ ] 技术指标替代原始K线
- [ ] 压缩订单簿数据（前3档+买卖比）
- [ ] 定时摘要而非实时全量（波动>1%或RSI超买/超卖触发）

### 提示词优化（DeepSeek建议二）
- [ ] 明确约束输出格式和长度
- [ ] 使用指令模板（场景专用）
- [ ] 设置max_tokens=10~50

### 调用策略优化（DeepSeek建议三）
- [ ] 降频轮询（1分钟1次，关键时段提高）
- [ ] 合并多个分析请求（多币种一次分析）
- [ ] 设置调用冷却时间（5分钟）

### 上下文管理（DeepSeek建议四）
- [ ] 无状态调用（不传历史上下文）
- [ ] 短期记忆仅保留关键决策
- [ ] 定时重置会话（每10次/每小时）

### 缓存机制（DeepSeek建议五）
- [ ] 本地缓存相同输入结果（1分钟缓存）
- [ ] 相似数据去重（价格变动<0.1%沿用上次）

### 本地计算
- [ ] 能在本地计算的，不调用API
- [ ] 使用基于规则的预测算法
- [ ] 批量生成预测结果
- [ ] 本地回填和验证
